# 2단계 신의 손 — 블랙홀 설계

- 작성일: 2026-07-14
- 상태: 승인됨 (구현 대기)
- 범위: 로드맵 **2단계(신의 손)** — 블랙홀, 질량 편집, 삭제
- 선행: [1단계 코어 설계](2026-07-14-space-sandbox-core-design.md) (구현 완료)

> 이 문서는 살아있는 설계 문서다. 구현 중 설계가 바뀌면 코드와 함께 이 문서를 갱신한다.

## 1. 목적과 원칙

블랙홀을 **재미 우선**으로 만들되, **모든 규칙이 실제 물리에서 파생되도록** 한다. 규칙을 밸런스를 위해 지어내지 않는다 — 실제 물리를 시뮬레이션 단위계로 번안하면 재미는 따라온다.

이 프로젝트는 샌드박스 장난감이고(1단계 설계 §1), 물리는 "그럴듯하고 안정적"이면 충분하다. 그러나 블랙홀은 **아무것도 안 하면 그냥 '검은 항성'이 된다** — 뉴턴 중력에서는 질량만 같으면 궤도가 동일하기 때문이다. 그러므로 "블랙홀이 무거운 항성과 무엇이 다른가"에 답하는 것이 이 설계의 전부다.

**답: 지평선 근처에 안정 궤도가 존재하지 않는다.** 일반상대성이론에서 `3r_s`(ISCO, 최내부 안정 원궤도) 안쪽에서는 어떤 속도로도 궤도를 유지할 수 없고 나선을 그리며 빨려든다. 뉴턴 중력에는 이런 것이 없다. 이 차이 하나가 게임 메커니즘이 된다.

## 2. 뿌리가 되는 상수 — 시뮬레이션 광속 `C`

블랙홀의 모든 것은 슈바르츠실트 반지름에서 나온다.

```
r_s = 2GM / c²
```

엔진에는 `G = 1`은 있지만 `c`가 없다. **`C`를 도입하면 나머지가 전부 파생된다.**

**`C = 25`** (`lib/sim/units.ts`). `G = 1`이므로:

| 파생값 | 공식 | M=3000 | M=10000 | M=50000 |
|---|---|---|---|---|
| 사건의 지평선 (= 렌더 반지름) | `r_s = 2M / C²` | 9.6 | 32 | 160 |
| **흡수 반경 (ISCO)** | `3 r_s = 6M / C²` | 28.8 | 96 | 480 |

`C`는 우리 우주의 자유 파라미터다. "우리 우주는 빛이 느리다"고 정한 것이고, 그 대가로 블랙홀이 손에 잡히는 크기가 된다. 실제 태양질량 블랙홀의 `r_s`는 3km로, 별 크기에 비하면 점에 불과하다 — 그대로 옮기면 화면에서 보이지도, 아무것도 삼키지도 못한다.

### 폭주 성장은 공짜로 따라온다

**`r_s`는 질량에 정비례한다.** 일반 천체의 반지름은 `∛m`으로 자란다(밀도 일정 가정, 1단계 설계 §4). 즉 블랙홀은 먹을수록 흡수 반경이 **선형으로** 커지고, 일반 천체보다 훨씬 빠르게 커진다. 위 표의 3000 → 50000 구간을 보면 흡수 반경이 29 → 480으로 뛴다. 시스템 전체를 삼킨다.

**폭주 성장을 규칙으로 만들 필요가 없다. 물리가 알아서 한다.**

## 3. 태어나는 법 — 두 경로

### 자동 붕괴 (물리)

질량이 **`COLLAPSE_MASS = 3000`** 을 넘으면 스스로 붕괴해 블랙홀이 된다.

찬드라세카르 한계(1.4 M☉, 전자 축퇴압의 한계)와 TOV 한계(~2-3 M☉, 중성자 축퇴압의 한계)의 번안이다. **"더 이상 버틸 브레이크가 없다"는 진짜 원리가 그대로 게임 규칙이 된다.**

항성 프리셋이 2000이므로 **항성 두 개를 충돌시키면 블랙홀이 된다.** 발견 가능하고 극적이다 — 병합으로 살을 찌우다 어느 순간 탁 하고 붕괴한다.

검사 시점: 매 서브스텝, 병합 직후. (병합이 질량을 바꾸는 유일한 경로이고, `setMass`/`spawn`도 다음 서브스텝에 잡힌다.)

### 강제 붕괴 (신의 손 치트)

정보 카드의 `블랙홀화` 버튼. **질량과 무관하게** 즉시 블랙홀로 만든다. 소행성도 가능하다.

물리적 근거가 없는 유일한 규칙이지만, **다음 절에서 물리가 스스로 벌을 준다.**

## 4. 호킹 증발 — 치트를 물리가 처벌한다

블랙홀은 질량을 잃는다.

```
dM/dt = -K / M²        (K = HAWKING_K = 0.2)
```

**작을수록 미친 듯이 빨리 증발한다** (증발 시간 ∝ M³).

| 블랙홀 질량 | 증발까지 (시뮬레이션 시간) |
|---|---|
| 1 (치트로 만든 소행성 블랙홀) | **약 2초** |
| 10 | 약 28분 |
| 100 | 약 19일 |
| 3000 (자연 붕괴 최소값) | 사실상 영원 |

질량이 `EVAPORATION_FLOOR = 0.01` 아래로 떨어지면 천체를 제거한다.

**이 규칙이 치트를 스스로 정리한다.** 버튼으로 소행성(질량 0.5)을 블랙홀로 만들면 흡수 반경이 `6×0.5/625 = 0.005`라 아무것도 못 먹고, 증발률이 폭발해 순식간에 사라진다. 밸런스를 위해 지어낸 제약이 아니라 **정확히 실제 물리가 말하는 바**다. 치트를 막을 필요가 없다.

증발은 시뮬레이션 시간 기준이다 — 배속을 올리면 빨리 증발한다.

## 5. 삼키는 법

일반 병합 조건은 `거리 < 반지름 합`이다(1단계 설계 §4). 블랙홀은 다르다.

**한쪽이 블랙홀이면, `거리 < ISCO(블랙홀)`일 때 속도와 무관하게 무조건 흡수된다.**

뉴턴 중력에서는 아무리 가까워도 빠르기만 하면 궤도를 돌 수 있다. 실제 블랙홀 근처에는 안정 궤도가 없다. 그 차이가 이 한 줄이다.

**중력 자체는 손대지 않는다.** 블랙홀은 같은 질량의 항성과 정확히 같은 힘으로 끌어당긴다. 이것이 정직함이자 재미다 — 멀리 있는 행성은 태연히 계속 공전하고, **위험은 오직 가까이 갔을 때만** 발생한다. "블랙홀은 우주 청소기"라는 가장 흔한 오해를 구현하지 않는 대신 진짜 무서움을 얻는다. (태양을 같은 질량의 블랙홀로 바꿔도 지구 궤도는 변하지 않는다.)

### 병합 규칙

| 상황 | 결과 |
|---|---|
| 블랙홀 + 일반 천체 | **블랙홀이 이긴다.** 질량이 작아도 결과는 블랙홀. |
| 블랙홀 + 블랙홀 | 블랙홀. 질량 합산. |
| 블랙홀의 반지름 | `r_s(m_total)` — 부피 보존(`∛(r₁³+r₂³)`)을 쓰지 않는다. |
| 위치·속도 | 기존 규칙 그대로 (운동량 보존). 단 [위치 고정](2026-07-14-space-sandbox-core-design.md)이 걸려 있으면 고정이 우선한다. |
| 정체성(id·색) | 블랙홀 쪽을 물려받는다 (질량 비교보다 우선). |

## 6. 보이는 법

- **검은 구.** 반지름 `r_s`, 색 `(0, 0, 0)`. `meshBasicMaterial`이라 조명이 없고, 블룸의 휘도 임계값(0.25)에도 걸리지 않아 완전히 검게 남는다. 기존 `Bodies` InstancedMesh를 그대로 쓴다 — 새 렌더 경로가 필요 없다.
- **강착원반.** `components/scene/AccretionDisks.tsx`. **안쪽 가장자리가 정확히 ISCO다** — 실제 강착원반도 그렇다. 그 안쪽엔 안정 궤도가 없어 물질이 머물 수 없기 때문이다. 따라서 **원반의 안쪽 테두리가 곧 죽음의 경계선**이며, 장식이 그대로 위험 표시기가 된다. 바깥 반경은 `2.5 × ISCO`. 황도면에 눕히고, 가산 혼합(additive)으로 빛나게 하며 블룸을 받는다. 단일 `InstancedMesh`(`RingGeometry`)로 draw call 1회.

## 7. 나머지 신의 손 — 질량 편집과 삭제

엔진에 `setMass`/`remove`가 이미 있다(1단계 설계 §10). 정보 카드에 UI만 붙인다.

- **질량 슬라이더** — 실시간 편집. 반지름이 따라 변하고, **3000을 넘기면 눈앞에서 붕괴한다.**
- **삭제 버튼** — 천체를 지운다. 선택도 함께 해제된다.

## 8. 엔진 표면 (추가되는 것)

```ts
// lib/sim/units.ts
export const C = 25;                    // 시뮬레이션 광속
export const COLLAPSE_MASS = 3000;      // 자동 붕괴 임계 질량
export const HAWKING_K = 0.2;           // 증발 계수
export const EVAPORATION_FLOOR = 0.01;  // 이 아래로 떨어지면 소멸
export function schwarzschildRadius(mass: number): number;  // 2M/C²
export function iscoRadius(mass: number): number;           // 3 r_s

// lib/sim/blackhole.ts (신규)
export function collapseAt(b: BodyBuffer, i: number): void;      // 강제 붕괴
export function applyCollapse(b: BodyBuffer): boolean;           // 임계 초과 자동 붕괴
export function applyHawking(b: BodyBuffer, dt: number): boolean; // 증발 + 소멸

// lib/sim/engine.ts
collapseToBlackHole(id: number): void;   // 치트 버튼용
isBlackHole(id: number): boolean;
```

`BodyType.BLACK_HOLE`(= 1)은 1단계에서 이미 자리를 잡아 뒀다. 새 플래그가 필요 없다.

`applyCollapse`/`applyHawking`이 `true`(천체 집합 변화)를 반환하면 엔진은 `accDirty`를 세운다.

## 9. 테스트 (Vitest)

| 테스트 | 검증 대상 |
|---|---|
| `r_s`가 질량에 정비례 | 폭주 성장의 근거 |
| ISCO = 3 r_s | |
| 임계 질량 초과 시 자동 붕괴 | 항성 2개 병합 → 블랙홀 |
| 임계 미달이면 붕괴하지 않음 | |
| `collapseToBlackHole`은 질량 무관하게 붕괴 | 치트 |
| 붕괴 시 반지름이 `r_s`로 바뀜 | |
| ISCO 안의 천체는 속도와 무관하게 흡수 | **이 설계의 핵심** — 원궤도 속도로 돌고 있어도 삼켜진다 |
| ISCO 밖의 천체는 정상 공전 | 중력은 그대로다 |
| 블랙홀 + 일반 = 블랙홀 | 작은 블랙홀이 항성을 먹어도 |
| 호킹 증발로 질량이 준다 | |
| 작은 블랙홀은 소멸한다 | 치트 자정 작용 |
| 큰 블랙홀은 사실상 증발하지 않는다 | |
| 결정론 유지 | 1단계 보장을 깨지 않았는가 |

## 10. 비목표

- **조석 파괴(스파게티화) / 파편화** — 1단계 비목표를 유지한다. 객체 수 폭발 위험이 크다.
- **중력 렌즈 효과** — 셰이더 작업이 크고, 이 톤에서 얻는 것보다 비용이 크다.
- **커 블랙홀(회전), 전하** — 무모 정리의 나머지 두 파라미터. 각운동량은 4단계 이후에나 의미가 있다.
- **강착원반의 물리 시뮬레이션** — 원반은 순수 시각 효과다. 실제로 물질이 원반을 이루며 도는 것은 시뮬레이션하지 않는다.
- **정보 역설** — 농담이다.
