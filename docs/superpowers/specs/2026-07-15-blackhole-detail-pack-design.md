# 블랙홀 디테일 팩 설계 — 광자 구 링 + 시간 지연

- 작성일: 2026-07-15
- 상태: 구현 완료 (2026-07-15)
- 범위: 블랙홀 후속 서브프로젝트 **C** — 가벼운 물리 디테일 팩. 광자 구 링(시각) + 정보 카드 시간 지연 표시.
- 선행: [블랙홀 이벤트→효과](2026-07-15-blackhole-effects-design.md) (구현 완료, A). [2단계 블랙홀](2026-07-14-black-hole-design.md) (구현 완료).

> 이 문서는 살아있는 설계 문서다. 구현 중 설계가 바뀌면 코드와 함께 이 문서를 갱신한다.

## 0. 로드맵에서의 위치

블랙홀 후속은 네 덩어리다: A(이벤트→효과, 완료), **C(이 문서, 디테일 팩)**, B(중력 렌즈, 이후), 조석 파괴(이후). C를 A 다음에 하는 이유는 값싼 재미이기 때문 — 둘 다 기존 패턴에 얹는 작은 추가라 큰 시각 투자(B) 전에 빠르게 얻는다.

## 1. 목적과 원칙

2단계 설계의 원칙을 잇는다: **규칙을 실제 물리에서 파생시킨다.** C가 더하는 두 디테일은 실재한다.

- **광자 구(1.5 r_s):** 이 반지름에서 빛은 블랙홀을 궤도로 돈다. 블랙홀 이미지(EHT·인터스텔라)에서 그림자를 감싸는 밝은 테두리가 이것이다. "여기서 빛조차 궤도에 갇힌다"는 표식이다.
- **중력 시간 지연:** 블랙홀에 가까울수록 시계가 느려진다(`√(1 − r_s/r)`). "가까이 가면 시간이 느려진다"는 직관을 숫자로 보여준다.

## 2. 시간 지연 공식 (`lib/sim/units.ts` — 순수 함수)

```
f = √(max(1 − r_s/r, 0))
```

`f`는 바깥 관찰자 기준 시계 속도 배율이다. 멀리(r→∞)면 `f→1`(지연 없음), 사건의 지평선(r=r_s)에서 `f=0`(시간 정지). `r ≤ r_s`면 0으로 클램프해 음수 sqrt를 막는다.

- `timeDilationAt(rs: number, r: number): number` — 위 식.
- `PHOTON_SPHERE_FACTOR = 1.5` — 광자 구 반지름의 r_s 배수.

**핵심 성질:** `r`을 r_s의 배수로 재면 `f`는 질량과 무관하다. 광자 구(1.5 r_s)에서 `f = √(1−1/1.5) ≈ 0.577`, ISCO(3 r_s)에서 `f = √(1−1/3) ≈ 0.816` — 모든 블랙홀에서 같은 값이다. 이것이 §4 (a)의 블랙홀 카드 값이 상수인 이유다.

## 3. 광자 구 링 (`components/scene/PhotonSpheres.tsx`)

블랙홀마다 `1.5 r_s`에 얇고 밝은 링. 단일 InstancedMesh, draw call 1회.

- **카메라를 향한다.** 매 프레임 각 링을 `dummy.lookAt(camera.position)`으로 정렬한다. RingGeometry는 법선이 로컬 +Z이고 `lookAt`이 +Z를 대상으로 향하게 하므로, 링이 카메라를 마주본다 — 블랙홀 이미지의 그 밝은 테두리처럼. 강착원반(황도면에 누운 평평한 링)과 확실히 구별된다.
- 광자 구(1.5 r_s)는 ISCO(3 r_s = 강착원반 안쪽 테두리)보다 **안쪽**이고 사건의 지평선(r_s = 검은 구 반지름)보다 **바깥**이다. 그래서 검은 구를 바로 감싸는 밝은 테두리로 보인다.
- 얇은 링: 단위 링 `ringGeometry(0.92, 1.0, 64)`를 `1.5 · schwarzschildRadius(mass)`로 스케일.
- 발광은 `meshBasicMaterial` + `toneMapped={false}` + Bloom. **`AdditiveBlending`을 쓰지 않는다**(강착원반이 유일 예외). 밝은 흰빛(광자 링은 밝다).
- `Bodies` **뒤에** 마운트(같은 프레임 상태를 읽는다), 엔진을 **읽기만** 한다. `useFrame` 안에서 할당하지 않는다(모듈 스코프 `dummy`). `AccretionDisks.tsx` 패턴을 따른다.
- 카메라는 `useFrame((state) => state.camera)`로 얻는다.

## 4. 시간 지연 표시 (`components/ui/BodyCard.tsx`)

정보 카드는 이미 선택 천체를 100ms 폴링해 표시한다. 여기에 시간 지연을 더한다.

**(a) 선택한 것이 블랙홀이면 — 기준 값(상수) 한 줄:**
기존 블랙홀 정보 블록(사건의 지평선·ISCO·증발까지)에 한 줄 추가:

```
시간 지연    광자 구 0.58× · ISCO 0.82×
```

값은 `timeDilationAt(rs, 1.5·rs)`와 `timeDilationAt(rs, 3·rs)`로 계산한다(상수지만 공식을 재사용해 일관성 유지). 질량과 무관한 교육적 값임을 라벨이 암시한다.

**(b) 선택한 것이 일반 천체이고 시뮬레이션에 블랙홀이 있으면 — 실제 겪는 값, 눈에 띌 때만:**
100ms 폴링 tick에서 모든 블랙홀을 훑어 **가장 강한 지연**(최소 `f`)을 주는 블랙홀을 찾는다. 그 `f`가 임계값(`TIME_DILATION_NOTICEABLE = 0.99`) **미만일 때만** 줄을 띄운다:

```
시간 지연    0.87×
```

- 멀어서 `f ≥ 0.99`(1% 미만 느려짐)면 줄을 숨긴다 — 밍밍한 "0.998×"로 카드를 어지럽히지 않는다.
- 블랙홀이 없으면 당연히 숨긴다.
- 여러 블랙홀은 **가장 강한 하나**만 본다(상대론적으로 단순 곱이 아니므로 합성하지 않는다 — §6 비목표).
- 폴링 tick의 블랙홀 훑기는 O(N)이고 10Hz라 비용이 무시할 만하다.

임계값 상수는 `BodyCard.tsx` 모듈 스코프에 둔다(시각 조정 대상).

## 5. 파일과 테스트

**수정:**
- `lib/sim/units.ts` — `timeDilationAt`, `PHOTON_SPHERE_FACTOR`.
- `components/ui/BodyCard.tsx` — (a) 블랙홀 기준 줄, (b) 일반 천체 실제 값(임계값 조건부, 최근접-강도 블랙홀 탐색).
- `components/scene/SpaceCanvas.tsx` — `<PhotonSpheres />`를 `Bodies` 뒤에 마운트.

**생성:**
- `components/scene/PhotonSpheres.tsx`.

**테스트(Vitest 대상은 `lib/sim`뿐):**
- `units.test.ts` — `timeDilationAt`: `r=r_s → 0`, `r ≤ r_s → 0`(클램프), `r→∞`(큰 r) → 1에 근접, `r=1.5·r_s → 0.577`, `r=3·r_s → 0.816`, 질량 무관성(다른 r_s라도 같은 배수면 같은 f).
- 광자 구 링(씬)과 카드 표시(DOM)는 Vitest 대상이 아니다 — 사람이 브라우저로 확인한다.

## 6. 비목표

- 실제 빛 휘어짐(중력 렌즈)은 B의 몫이다. 광자 구 링은 "여기서 빛이 궤도를 돈다"는 표식일 뿐, 배경을 휘지 않는다.
- 여러 블랙홀의 시간 지연 합성(상대론적으로 단순 곱이 아님) — 가장 강한 하나만 표시한다.
- 시간 지연이 시뮬레이션 물리에 되먹임되지 않는다(천체가 실제로 느리게 움직이지 않는다). 순수 표시다.
- 광자 구가 흡수·충돌 판정에 관여하지 않는다(흡수는 여전히 ISCO). 순수 시각 표식이다.

## 7. 조정 가능한 숫자

- `TIME_DILATION_NOTICEABLE = 0.99` — 일반 천체 카드에 시간 지연 줄을 띄우는 임계. 낮추면 아주 가까울 때만 뜬다.
- 광자 구 링의 두께(단위 링 0.92~1.0), 색, 밝기 — 시각 조정 대상.

## 8. 사람의 브라우저 확인 (자동화 불가)

- 블랙홀 주위에 검은 구를 감싸는 밝은 카메라-정면 링(광자 구)이 보이는가. 카메라를 돌려도 항상 정면인가. 강착원반과 구별되는가.
- 링이 강착원반 안쪽(검은 구와 원반 사이)에 앉는가.
- 블랙홀을 선택하면 카드에 "시간 지연: 광자 구 0.58× · ISCO 0.82×"가 뜨는가.
- 일반 천체를 블랙홀 가까이 두고 선택하면 시간 지연이 1 아래로 떨어져 표시되고, 멀리 두면 줄이 사라지는가.
