- 상태: 승인됨 (구현 대기)
- 범위: 시각 효과. 강착원반에서 관찰자를 향해 도는 쪽을 밝고 푸르게, 멀어지는 쪽을 어둡고 붉게 표시(상대론적 도플러 비밍). **표시만 바꾸며 엔진·물리는 건드리지 않는다.**

## 1. 목적

지금 강착원반(`AccretionDisks.tsx`)은 균일한 주황색 링이다. 실제 블랙홀 사진(EHT)·인터스텔라의 상징적인 **밝은 초승달**은 도플러 비밍 — 원반 물질이 광속의 상당 비율로 돌기 때문에, 관찰자를 향해 다가오는 쪽이 밝고 푸르게, 멀어지는 쪽이 어둡고 붉게 보이는 현상이다. 이 한 가지로 원반이 단숨에 "진짜"처럼 보인다.

**핵심 성질 — 시점 의존:** 밝은 쪽은 원반의 공전 방향과 시선 방향의 관계로 정해진다. 카메라를 돌리면 밝은 초승달도 따라 움직여야 한다. 고정 그라디언트로 칠하면 카메라 회전 시 어색하다. 이 시뮬은 계속 궤도 회전하며 보므로 시점 의존이 필수다.

## 2. 모델 (간이 상대론)

원반은 황도면(XZ)에서 +Y축을 중심으로 돈다고 가정한다(엔진의 블랙홀엔 스핀이 없다 — 스핀은 확장 3/3 커 블랙홀에서). 링 위 각 점에서:

- **공전 속도 방향** `v̂` — 그 점의 방사 방향에 수직한 접선. 모든 원반이 같은 평면·같은 회전 방향이므로 `v̂`는 정점마다 **상수**다(월드 기준, 한 번만 선계산).
- **시선 방향** `ŝ` — 블랙홀 중심 → 카메라. 원반 반지름이 카메라 거리보다 작으므로 원반당 하나로 근사(정점마다 다시 안 구함).
- **비밍 계수** `beam = v̂ · ŝ ∈ [−1, 1]`. `+`(다가옴) → 밝고 푸르게, `−`(멀어짐) → 어둡고 붉게.

이 모델은 시야각을 자연스럽게 처리한다: 정면(카메라가 극에서 봄)이면 `ŝ ≈ +Y`라 모든 `v̂·ŝ ≈ 0` → 비대칭 없음(정면 원반은 지금처럼 균일). 측면(카메라가 황도면)이면 `v̂·ŝ`가 −1~1로 벌어짐 → 강한 초승달. 실제 물리와 정성적으로 일치.

**색·밝기 매핑** (튜닝 상수 `BEAM_STRENGTH`):
```
boost = max(0.15, 1 + BEAM_STRENGTH · beam)   // 멀어지는 쪽은 어둡게(하한 0.15로 완전히 안 사라지게)
r = 1.0 · boost
g = (0.55 + 0.35 · beam) · boost              // 다가올수록 노랗게→희게, 멀어질수록 붉게
b = (0.20 + 0.60 · max(0, beam)) · boost       // 다가오는 쪽만 푸른기, 멀어지는 쪽은 붉게 유지
```
`toneMapped={false}` + 가산 혼합 + 블룸이라 `boost`로 1을 넘긴 값은 다가오는 쪽에서 빛나는 초승달이 된다. `beam=0`이면 기존 주황색(≈`#ff9d3c`)으로 수렴한다.

## 3. 렌더링 접근 (A안: 정점 색 + 메시 풀)

셰이더를 새로 들이지 않고, 이 코드베이스의 관습(`meshBasicMaterial` + 블룸, "자기 `BufferAttribute`를 ref로 들고 매 프레임 갱신")을 그대로 쓴다.

- 현재는 모든 블랙홀 원반이 **하나의 `InstancedMesh`**다. 인스턴싱은 정점 색을 인스턴스별로 못 준다 → **메시 풀**로 바꾼다.
- `<group>` 아래 `MAX_DISKS`(=24)개의 `<mesh>`를 미리 만든다. 각 메시는 자기 `RingGeometry`와 자기 `color` `BufferAttribute`를 가진다. `material.vertexColors = true`.
- 공전 접선 `v̂`는 모든 원반이 공유하므로 템플릿 지오메트리에서 **한 번만** 선계산한다.
- `useFrame`마다: 블랙홀을 순회하며 활성 원반에 하나씩 배정 → 위치·회전(−90°X로 황도면)·크기(ISCO)를 세팅하고, `ŝ`를 구해 정점 색 배열을 채우고 `colorAttr.needsUpdate = true`. 남는 풀 메시는 `visible = false`. React 리렌더 0회(엔진 버퍼를 직접 읽음).
- 지오메트리는 `useMemo`로 만들고 언마운트 `useEffect`에서 `dispose()`.

**받아들인 한계:** 동시 블랙홀이 `MAX_DISKS`(24)를 넘으면 초과분은 원반이 안 그려진다(흡수 물리는 그대로). 블랙홀을 스물넷 넘게 동시에 두는 극단적 상황에서만이고, 그땐 개별 원반을 감상할 상황이 아니다. draw call은 원반 수만큼(보통 1~3), 기존 1회에서 늘지만 무시할 수준.

## 4. 파일

**수정:**
- `components/scene/AccretionDisks.tsx` — 인스턴싱 → 메시 풀 + 정점 색 비밍.

생성·삭제 파일 없음. 다른 씬 객체(`GravitationalLensing`, `PhotonSpheres`, `EffectsController`)는 건드리지 않는다.

## 5. 조정 가능한 숫자

- `BEAM_STRENGTH`(기본 1.6) — 밝은/어두운 쪽 대비. 사람이 브라우저에서 맞춘다.
- `MAX_DISKS`(24) — 동시 비밍 원반 상한.
- 색 매핑 계수(g·b의 기울기, boost 하한 0.15) — 초승달의 색감·잔광. 가독성/미감 조정 대상.

## 6. 비목표

- **엔진·물리 변경 없음.** 궤도·흡수·질량 전부 불변. 색만 바뀐다.
- **원반 회전 애니메이션·질감 없음.** 균일 링이라 회전이 눈에 안 보이고, 비밍 초승달은 시점에 고정된다. 소용돌이 줄무늬는 별도(스코프 밖).
- **정확한 상대론 파형 아님.** `D = 1/(γ(1−β cosθ))`의 정식 `I ∝ D^(3+α)`를 쓰지 않고 선형 근사(`1 + BEAM·beam`)로 감을 준다. C가 비물리적인 이 우주(§units)에 맞춘 시각 튜닝이다.
- **스핀 의존 없음.** 원반은 +Y축 고정 회전. 스핀은 확장 3/3(커 블랙홀).

## 7. 사람의 브라우저 확인 (자동화 불가)

- 블랙홀 프리셋을 불러와 측면에서 보면, 원반 한쪽이 밝고 푸른 초승달, 반대쪽이 어둡고 붉게 보이는가.
- 카메라를 원반 둘레로 돌리면 밝은 초승달이 **시점을 따라 움직이는가**(고정 그라디언트가 아님).
- 원반을 거의 정면(위)에서 보면 비대칭이 사라지고 균일한 주황에 가까워지는가.
- 블랙홀 여러 개를 만들어도 각자 자기 초승달을 그리는가(풀 상한 내에서).
- 프레임이 유지되는가(보통 씬).
