# 중력 렌즈 설계 — 화면 공간 후처리 왜곡

- 작성일: 2026-07-15
- 상태: 구현 완료 (2026-07-15)
- 범위: 블랙홀 후속 서브프로젝트 **B** — 중력 렌즈. 블랙홀 뒤의 빛(배경별·천체·강착원반·광자 구 링)이 휘어 보이는 전체 화면 왜곡.
- 선행: [블랙홀 디테일 팩](2026-07-15-blackhole-detail-pack-design.md) (구현 완료, C — 광자 구 링 + 시간 지연). [블랙홀 이벤트→효과](2026-07-15-blackhole-effects-design.md) (구현 완료, A). [2단계 블랙홀](2026-07-14-black-hole-design.md) (구현 완료).

> 이 문서는 살아있는 설계 문서다. 구현 중 설계가 바뀌면 코드와 함께 이 문서를 갱신한다.

## 0. 로드맵에서의 위치

블랙홀 후속은 네 덩어리다: A(이벤트→효과, 완료), C(디테일 팩, 완료), **B(이 문서, 중력 렌즈)**, 조석 파괴(이후). B는 예고된 "큰 시각 투자"다 — A·C가 기존 패턴에 얹는 작은 추가였다면, B는 커스텀 후처리 셰이더라는 새 축을 연다. 그래서 값싼 것(A·C)을 먼저 하고 B를 뒤에 뒀다.

## 1. 목적과 원칙

블랙홀 설계의 원칙을 잇는다: **규칙을 실제 물리에서 파생시킨다.** 질량 근처를 지나는 빛은 휜다(`α = 2·r_s / b`). B는 이 휘어짐을 화면에 보여준다 — 블랙홀 뒤 배경이 그림자를 감싸며 링으로 모이는, 인터스텔라·EHT의 그 상징적 모습.

- **전체 화면 왜곡:** 배경별뿐 아니라 블랙홀 뒤로 지나가는 행성·항성, 그리고 강착원반·광자 구 링까지 프레임버퍼 전체가 휜다. 강착원반이 블랙홀 위로 아치처럼 감기는 룩이 이 방식에서 공짜로 나온다.
- **표시 전용:** 렌즈는 시뮬레이션 물리에 되먹임하지 않는다. 흡수·충돌 판정은 여전히 ISCO 기준이고 천체는 실제로 휘어 이동하지 않는다. 순수 시각 효과다(A·C와 같은 결정론 원칙).

## 2. 휘어짐 공식 (`lib/sim/units.ts` — 순수 함수)

```
α = 2·r_s / b
```

`α`는 충돌 파라미터 `b`(빛이 블랙홀 중심을 스치는 최소 거리)에서 빛이 휘는 각의 근사다. 약한장(b ≫ r_s) 근사이며, 이 프로젝트의 다른 물리 상수처럼 교육적으로 옳은 스케일을 준다.

- `lensDeflection(rs: number, b: number): number` — 위 식. `b ≤ 0`이면 0을 반환해 0 나눗셈을 막는다(물리적으로 b는 양수).

**핵심 성질:**
- `b = r_s`에서 `α = 2`, `b = 2·r_s`에서 `α = 1` — 거리가 2배면 휘어짐이 절반.
- `b`가 커지면 `α → 0` — 멀리서는 안 휜다.
- 같은 `b`라도 `r_s`(질량)가 클수록 `α`가 크다 — 큰 블랙홀이 더 크게 휜다.

이 함수는 셰이더 왜곡 강도의 물리적 근거다. 셰이더는 화면 거리 `d`에 대해 변위 크기를 `∝ r_s² / d`로 주어(가까울수록 강하게 중심으로 당김) 이 1/거리 감쇠를 재현한다. 아인슈타인 링 반지름은 대략 `θ_E ∝ √r_s`로, 큰 블랙홀일수록 링이 크다.

## 3. 렌즈 이펙트 (`components/scene/GravitationalLensing.tsx`)

기존 `EffectComposer` 파이프라인에 커스텀 후처리 이펙트 하나를 얹는다. `postprocessing`의 `Effect`를 상속한 서브클래스(`LensingEffectImpl`)를 `useMemo`로 만들고 `<primitive object={effect} dispose={null} />`로 `<EffectComposer>` 자식으로 마운트한다. 셰이더 진입점은 `void mainUv(inout vec2 uv)`이며, `blendFunction`은 `NORMAL`(기본 `SCREEN`이면 원본과 합성돼 화면이 밝아진다). `postprocessing`의 `ShockWave`(한 점 중심 방사형 화면 왜곡, 카메라로 3D 위치 투영)가 같은 기법의 선례다.

**데이터 흐름 (매 프레임):**
1. R3F `useFrame`에서 카메라(`state.camera`)와 엔진에 접근한다.
2. 각 블랙홀의 월드 좌표를 카메라로 투영해 **화면 좌표(NDC/UV)** 와 **화면상 사건의 지평선 반지름**(월드 r_s를 화면 스케일로 환산)을 구한다.
3. 화면상 겉보기 크기가 큰 순으로 최대 `MAX_LENSES = 4`개만 골라 uniform 배열(중심 UV, 화면 반지름/강도)에 채운다. 활성 렌즈 개수도 uniform으로 넘긴다.
4. 프래그먼트 셰이더가 각 픽셀에서 활성 렌즈를 순회한다.

**프래그먼트 셰이더 (개념):**
- 픽셀 UV와 렌즈 중심의 차 벡터를 **화면비 보정**(가로세로 왜곡 방지) 후 거리 `d`를 구한다.
- 변위 = `(중심 방향 단위 벡터) · (renderRadius² / d)` 꼴로 배경 샘플 좌표를 중심 쪽으로 당긴다 → 빛이 모여 링 생성. 여러 렌즈는 변위를 누적한다.
- **영향 반경:** 왜곡은 사건의 지평선 반지름의 `INFLUENCE_RADII`배(기본 6, = 지름의 3배)까지만 미치고 그 밖은 0이다. 경계에서 `smoothstep`으로 부드럽게 감쇠해 화면 전체로 새지 않고 블랙홀 근처로 국소화된다. 중심 부근은 감쇠 계수가 거의 1이라 강한 렌즈 형태를 유지한다.
- 검은 사건의 지평선 그림자는 별도로 합성하지 않는다. 이미 렌더된 블랙홀 구체가 프레임버퍼에 있어 UV 왜곡과 함께 휘며 그림자가 된다. 셰이더 진입점은 `mainUv(inout vec2 uv)`이며, UV를 렌즈 중심으로 당기면 프레임워크가 그 좌표로 입력을 샘플링한다.
- 최종적으로 왜곡된 UV로 입력 프레임버퍼를 샘플링한다.

**규칙 준수:**
- 엔진을 **읽기만** 한다. 물리에 되먹임 없음.
- `useFrame`/`update` 안에서 할당하지 않는다 — 모듈 스코프 재사용 객체(`THREE.Vector3` 등)로 투영. `Bodies.tsx`/`PhotonSpheres.tsx` 패턴을 따른다.
- 깊이 버퍼를 쓰지 않는다(전체 화면 왜곡이므로 불필요). 프레임버퍼 전체를 휜다.

**마운트 위치:** `SpaceCanvas.tsx`의 `<EffectComposer>` 안, `<Bloom>` **앞에**. 렌즈가 먼저 화면을 휘고, 그 위에 블룸이 발광을 얹는다.

## 4. 성능

- 셰이더 렌즈 순회는 고정 상한 `MAX_LENSES = 4`. 블랙홀이 5개 이상이면 화면상 작고 먼 것은 왜곡에서 제외한다 — 시각적으로 무시할 만하고 프래그먼트 비용을 상한 짓는다.
- CPU 측 투영은 O(블랙홀 수)이고 프레임당 한 번이라 비용이 무시할 만하다.
- 블랙홀이 없으면 활성 렌즈 0 — 셰이더는 즉시 원본 UV를 반환해 사실상 no-op(이펙트를 언마운트하지 않고 유지).

## 5. 파일과 테스트

**수정:**
- `lib/sim/units.ts` — `lensDeflection(rs, b)`.
- `components/scene/SpaceCanvas.tsx` — `<GravitationalLensing />`를 `<Bloom>` 앞에 마운트.

**생성:**
- `components/scene/GravitationalLensing.tsx`.

**테스트(Vitest 대상은 `lib/sim`뿐):**
- `units.test.ts` — `lensDeflection`: `b = r_s → α = 2`, `b = 2·r_s → α = 1`, 큰 `b`(예: 100·r_s) → 0에 근접, 같은 `b`에서 r_s 클수록 α 큼, `b ≤ 0 → 0`(0 나눗셈 방지).
- 셰이더 왜곡(화면)은 Vitest 대상이 아니다 — 사람이 브라우저로 확인한다.

**구현 시 소스 확인.** `postprocessing`의 커스텀 `Effect` 정확한 API(생성자 uniform 배열 전달 방식, `update(renderer, inputBuffer, deltaTime)` 훅에서 카메라 접근, `wrapEffect` R3F 결합)를 계획·구현 단계에서 `node_modules` 실물로 확인한다. 버전이 훈련 데이터와 다를 수 있다(AGENTS.md 원칙).

## 6. 비목표

- **실제 광선 추적/다중 상.** 화면 공간 근사다. 뒤에 완전히 가려진 천체가 링 반대편에 두 번 보이는 진짜 중복상(multiple imaging)은 재현하지 않는다 — 아치·링 모양만 흉내 낸다.
- **깊이 기반 선택적 왜곡.** 전체 화면 왜곡으로 결정했다. 가까운 전경 천체도 함께 휜다(의도한 인터스텔라 룩).
- **물리 되먹임.** 렌즈는 표시 전용. 흡수는 여전히 ISCO, 천체는 실제로 휘어 이동하지 않는다.
- **다중 렌즈 정밀 합성.** 여러 블랙홀의 변위를 단순 누적한다(상대론적으로 엄밀하지 않지만 시각적으로 충분).

## 7. 조정 가능한 숫자

- `MAX_LENSES = 4` — 동시에 왜곡에 넣는 블랙홀 상한. 성능/시각 조정 대상.
- `INFLUENCE_RADII = 6` — 왜곡이 미치는 영향 반경(사건의 지평선 반지름의 배수, = 지름의 3배). 키우면 더 넓게 휘고, 줄이면 블랙홀에 더 바싹 붙는다.
- 변위 강도 계수(`renderRadius² / d`의 배율), 링 반지름·감쇠 곡선 — 시각 조정 대상.
- 사건의 지평선 그림자 반지름의 화면 환산 배율 — 검은 원반 크기 조정.

## 8. 사람의 브라우저 확인 (자동화 불가)

- 블랙홀 뒤 배경별이 그림자를 감싸며 링으로 휘어 모이는가.
- 카메라를 돌려도 왜곡 중심이 블랙홀을 따라가는가.
- 행성이 블랙홀 뒤로 지날 때 상이 아치처럼 늘어나는가.
- 강착원반/광자 구 링이 블랙홀 위로 감겨 인터스텔라 룩이 나는가.
- 큰 블랙홀이 작은 블랙홀보다 더 크게 휘는가(질량 연동).
- 블랙홀이 없을 때 화면이 왜곡 없이 정상인가.
- 블랙홀이 5개 이상일 때 프레임이 유지되는가(작고 먼 것 제외 확인).
