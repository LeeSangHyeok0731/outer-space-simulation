# GitHub Labels Reference

Select 1-2 labels from the PR-eligible list below. Use the exact repository label names.
The repository currently uses GitHub default labels only.

## PR-Eligible Labels

| Label           | When to Use                                        |
| --------------- | -------------------------------------------------- |
| `bug`           | Bug fix PR                                         |
| `enhancement`   | New feature, improvement, or harness capability    |
| `documentation` | Documentation-only changes                         |

## Off-Limits Labels

| Label              | Reason                                            |
| ------------------ | ------------------------------------------------- |
| `duplicate`        | Issue triage state, not a PR label                |
| `good first issue` | Issue onboarding label                            |
| `help wanted`      | Issue triage label                                |
| `invalid`          | Issue triage state                                |
| `question`         | Issue type, not a PR label                        |
| `wontfix`          | Resolution state                                  |

## Quick Decision

```text
Bug fix? -> bug
Docs only? -> documentation
Anything else (feature, refactor, chore, UI)? -> enhancement
```

If a needed label does not exist, skip labels instead of inventing one.
